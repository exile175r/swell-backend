# STT 백엔드 실시간 연동 완료 보고서

요청하신 대로 백엔드에서 실시간으로 `faster-whisper` 라이브러리가 실행되고 STT가 진행되는 구조를 완성했습니다.

## 주요 구현 사항

### 1. 고성능 Python STT 엔진 (`src/services/stt_stream.py`)
- `faster-whisper` (tiny 모델)를 사용하여 CPU에서도 빠른 음성 인식이 가능하도록 최적화했습니다.
- 모델 로딩을 개별 요청마다 하지 않고, 엔진 실행 시 1회만 로드하여 응답 속도를 극대화했습니다.
- 실시간성을 위해 `beam_size`를 조정하였습니다.

### 2. 실시간 소켓 브릿지 (`src/services/stt.socket.ts`)
- `Socket.io`를 통해 클라이언트가 보내는 오디오 청크를 실시간으로 Python 엔진에 전달합니다.
- **프로세서 관리**: 소켓 연결 시 Python 프로세스를 유지하고, 연결 해제 시 자원을 자동으로 정리합니다.
- **실시간 피드백**: 변환된 텍스트가 즉시 클라이언트로 전송됩니다.

### 3. API 컨트롤러 및 Swagger 연동
- 기존의 Mock 응답을 제거하고, 실제 STT 엔진과 연동할 수 있는 구조로 변경했습니다.
- Swagger 문서를 최신화하여 실시간 STT 엔진 사양이 정확히 노출되도록 조치했습니다.

### 4. 유저 차단(Block) 기능 구현
- **차단 관계 관리**: `Block` 모델을 추가하여 유저 간의 차단 데이터를 영구 보관합니다.
- **콘텐츠 보안 필터**: 게시글 목록 및 프로필 조회 시 차단된 유저의 정보를 자동으로 필터링하는 로직을 통합했습니다.

### 5. 신고(Report) 시스템 구축
- **다각도 신고 접수**: 유저, 게시글, 댓글 등 모든 주요 콘텐츠에 대한 신고 체계를 마련했습니다.
- **누적 신고 추적**: 신고 시 해당 유저의 과거 신고 이력을 실시간으로 합산하여 관리자에게 제공합니다.
- **관리자 라이프사이클**: [경고], [정지], [반려] 등의 조치 프로세스를 API 레벨에서 완비했습니다.
### 6. 콘텐츠 필터링(Content Filtering) 시스템
- **비속어 마스킹**: 전역 유틸리티(`filter.ts`)를 통해 모든 게시글과 댓글의 비속어를 **게시(저장) 시점에** `***` 처리합니다.
- **원본 보존 조회**: 작성자 본인은 마이페이지 등에서 수정 없이 원본을 볼 수 있고, 타인에게는 필터링된 버전만 노출되는 이중 저장 구조를 구축했습니다.
- **블랙리스트 기반 차단**: 무분별한 링크 차단 대신, 관리자가 등록한 특정 도메인이나 단어만 정밀하게 필터링합니다.
- **도배 및 악성 게시물 방지**: 1분당 게시물 수 제한(Rate Limit) 및 비정상적인 문자 반복(예: ㅋㅋㅋㅋ...)을 감지하여 사전에 차단합니다.

## 테스트 및 검증 결과

### 엔진 설치 확인
- 백엔드 서버 환경에서 `faster-whisper` 라이브러리가 정상 설치되어 있음을 확인했습니다.

### 프로세스 동작 확인
- 소켓 연결 시 Python 프로세스가 `-u` (unbuffered) 옵션으로 실행되어 즉각적인 데이터 파이프라이닝이 이루어짐을 확인했습니다.

> [!IMPORTANT]
> 이제 백엔드에서는 프론트엔드의 호출에 따라 즉시 실무 수준의 STT 엔진이 구동됩니다. 프론트엔드 개발자분은 기존 소켓 이벤트를 그대로 사용하시되, 실시간 피드백을 바로 수신하실 수 있습니다.
